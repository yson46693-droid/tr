# خطة تنفيذ وضع الصيانة وحساب المطور

## نظرة عامة

سيتم إضافة نظام وضع الصيانة الذي يمنع جميع المستخدمين (عدا حساب المطور) من استخدام النظام عند تفعيله، مع عرض تنبيه منبثق ثابت.

## التغييرات المطلوبة

### 1. تحديث قاعدة البيانات

#### أ. تعديل جدول المستخدمين لإضافة دور "developer"

- تحديث `users.role` enum لإضافة `'developer'` إلى القائمة: `'accountant','sales','production','manager','developer'`
- الملف: `database/schema.sql`
- SQL Query:
```sql
ALTER TABLE `users` MODIFY COLUMN `role` enum('accountant','sales','production','manager','developer') NOT NULL;
```


#### ب. إضافة إعداد وضع الصيانة

- إضافة سجل في `system_settings` لوضع الصيانة (القيمة الافتراضية: 'off')
- الملف: `database/schema.sql`
- SQL Query:
```sql
INSERT INTO `system_settings` (`key`, `value`) VALUES ('maintenance_mode', 'off') ON DUPLICATE KEY UPDATE `key`='maintenance_mode';
```


#### ج. إنشاء حساب المطور

- SQL Query لإنشاء حساب المطور (يتم تنفيذه يدوياً):
```sql
INSERT INTO `users` (`username`, `email`, `password_hash`, `role`, `full_name`, `status`) 
VALUES ('developer', 'developer@company.com', '$2y$10$92IXUNpkjO0rOQ5byMi.Ye4oKoEa3Ro9llC/.og/at2.uheWG/igi', 'developer', 'حساب المطور', 'active');
```

- كلمة المرور الافتراضية: `password` (يجب تغييرها)

### 2. تعديل نظام المصادقة (`includes/auth.php`)

#### أ. إضافة دوال للتحقق من وضع الصيانة

- `isMaintenanceMode()`: للتحقق من حالة وضع الصيانة من `system_settings`
- `isDeveloper()`: للتحقق إذا كان المستخدم الحالي له دور 'developer'
- `checkMaintenanceMode()`: للتحقق من وضع الصيانة ورفض الوصول للمستخدمين العاديين

#### ب. تعديل دالة `requireLogin()`

- إضافة فحص وضع الصيانة بعد التحقق من تسجيل الدخول
- إذا كان وضع الصيانة مفعلاً والمستخدم ليس مطوراً، منع الوصول
- حفظ حالة وضع الصيانة في `$_SESSION` للوصول السريع

### 3. إنشاء صفحة إعدادات النظام (`modules/manager/system_settings.php`)

#### أ. الواجهة

- صفحة في لوحة المدير لإدارة إعدادات النظام
- زر تبديل لوضع الصيانة (Toggle Switch)
- عرض حالة وضع الصيانة الحالية

#### ب. معالجة POST

- معالجة طلب تفعيل/تعطيل وضع الصيانة
- تحديث قيمة `maintenance_mode` في `system_settings`
- تسجيل العملية في `audit_log`

### 4. إضافة رابط في القائمة الجانبية

#### تعديل `templates/sidebar_manager.php`

- إضافة عنصر قائمة جديد "إعدادات النظام" يشير إلى `manager.php?page=system_settings`

### 5. تعديل لوحة المدير (`dashboard/manager.php`)

- إضافة معالجة لصفحة `system_settings` لتضمين `modules/manager/system_settings.php`

### 6. إضافة واجهة التنبيه في الصفحات

#### أ. إنشاء مكون JavaScript/CSS

- Modal/Overlay ثابت لا يمكن إغلاقه
- الرسالة: "التطبيق تحت الصيانة في الوقت الحالي برجاء إعادة المحاولة في وقت لاحق"
- منع جميع التفاعلات (pointer-events: none على المحتوى)
- عرض Modal فوق كل المحتوى

#### ب. إضافة في `templates/header.php` أو `templates/footer.php`

- JavaScript للتحقق من وضع الصيانة عبر AJAX
- إذا كان الوضع مفعلاً والمستخدم ليس مطوراً، عرض Modal
- منع أي تفاعلات مع الصفحة

### 7. إنشاء API endpoint للتحقق من وضع الصيانة

#### ملف جديد: `api/check_maintenance_mode.php`

- Endpoint يعيد حالة وضع الصيانة
- يستخدم `isMaintenanceMode()` من `auth.php`
- يعيد JSON: `{"maintenance_mode": "on"/"off"}`

### 8. تعديل دالة تسجيل الدخول (`includes/auth.php` - `login()`)

- بعد تسجيل الدخول بنجاح، التحقق من وضع الصيانة
- إذا كان مفعلاً والمستخدم ليس مطوراً، إرجاع رسالة خطأ وعدم السماح بتسجيل الدخول

## ملفات سيتم تعديلها

1. `database/schema.sql` - تحديث بنية قاعدة البيانات
2. `includes/auth.php` - إضافة دوال وضع الصيانة وتعديل `requireLogin()` و `login()`
3. `modules/manager/system_settings.php` - **ملف جديد** - صفحة إعدادات النظام
4. `templates/sidebar_manager.php` - إضافة رابط إعدادات النظام
5. `dashboard/manager.php` - إضافة معالجة لصفحة system_settings
6. `templates/header.php` أو `templates/footer.php` - إضافة JavaScript و Modal لوضع الصيانة
7. `api/check_maintenance_mode.php` - **ملف جديد** - API للتحقق من وضع الصيانة

## ملاحظات تنفيذية

- حساب المطور يجب أن يكون له دور `'developer'` في قاعدة البيانات
- وضع الصيانة لا يؤثر على حساب المطور أبداً
- التنبيه يكون ثابتاً ولا يمكن إغلاقه
- جميع المستخدمين (عدا المطور) سيتم منعهم من استخدام أي صفحة عند تفعيل وضع الصيانة
- يجب التأكد من أن وضع الصيانة يتم التحقق منه في جميع نقاط الدخول (login, requireLogin, API endpoints)
