# حل مشكلة "Too many connections" في MySQL

## المشكلة
عند ظهور رسالة الخطأ: `MySQL connection error: Too many connections`

هذا يعني أن عدد الاتصالات المتزامنة بقاعدة البيانات وصل إلى الحد الأقصى المسموح به في MySQL.

## الحلول المنفذة في الكود

### 1. تحسين إدارة الاتصالات
- تم إضافة آلية لإعادة المحاولة عند حدوث "Too many connections"
- إغلاق تلقائي للاتصال عند انتهاء الطلب
- التحقق من حالة الاتصال قبل إعادة الاستخدام
- تعيين timeout للاتصالات لتقليل الاتصالات المعلقة

### 2. تقليل الاتصالات المعلقة
- تم تعيين `wait_timeout` و `interactive_timeout` إلى 5 دقائق
- إغلاق الاتصالات تلقائياً عند انتهاء الطلب

## الحلول الإضافية

### الحل السريع (مؤقت)
قم بتشغيل الاستعلام التالي في MySQL:
```sql
SET GLOBAL max_connections = 500;
```

### الحل الدائم

#### للأجهزة المحلية (XAMPP/WAMP)

1. افتح ملف `my.ini` (Windows) أو `my.cnf` (Linux/Mac) في مجلد MySQL
2. ابحث عن قسم `[mysqld]`
3. أضف أو عدل الإعدادات التالية:
```ini
[mysqld]
max_connections = 500
wait_timeout = 300
interactive_timeout = 300
```
4. أعد تشغيل MySQL

#### للسيرفرات البعيدة (cPanel/phpMyAdmin)

1. افتح phpMyAdmin
2. اذهب إلى "Variables" في القائمة العلوية
3. ابحث عن `max_connections`
4. اضغط على "Edit" وقم بزيادة القيمة
5. احفظ التغييرات

**ملاحظة:** قد تحتاج صلاحيات خاصة (SUPER) لتعديل `max_connections`. في حالة عدم توفر الصلاحيات، اتصل بمزود الاستضافة.

## التحقق من الحل

قم بتشغيل الاستعلامات التالية للتحقق:
```sql
-- عرض الحد الحالي للاتصالات
SHOW VARIABLES LIKE 'max_connections';

-- عرض عدد الاتصالات النشطة حالياً
SHOW STATUS LIKE 'Threads_connected';

-- عرض أكبر عدد اتصالات تم استخدامه
SHOW STATUS LIKE 'Max_used_connections';
```

## نصائح إضافية

1. **إغلاق الاتصالات غير المستخدمة:** تأكد من أن جميع الاتصالات تُغلق بشكل صحيح
2. **استخدام Connection Pooling:** إذا كان ذلك متاحاً في بيئتك
3. **مراقبة الاتصالات:** راقب عدد الاتصالات النشطة بانتظام
4. **تحسين الكود:** تجنب فتح اتصالات متعددة في نفس الطلب
